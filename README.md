# Waybills — сборка и печать накладных Kaspi Доставки

Веб-сервис + агент для склада. Забирает из Kaspi Shop API заказы в статусе "готов к передаче курьеру", сортирует их для сборщиков (группы A/B/C × ходовые товары), готовит PDF под термо-этикетку 75×120 мм, разбивает под рулоны, отправляет на печать через агент на складском ноуте.

## Архитектура

```
┌───────────────────┐            ┌───────────────────┐
│   Веб-UI          │            │  Kaspi Shop API   │
│   (браузер)       │            │  /orders          │
└──────┬────────────┘            └──────▲────────────┘
       │ HTTPS                          │
┌──────▼──────────────────────┐         │
│  FastAPI на VPS (:8090)     │─────────┘
│  ─ парсит Kaspi             │
│  ─ генерит PDF (75×120мм)   │
│  ─ SQLite + очередь задач   │
└──────┬──────────────────────┘
       │ /agent/next-task
       │ Bearer <AGENT_TOKEN>
┌──────▼────────────┐  ┌─────────┐  ┌────────┐
│  Агент (склад)    │→ │ Sumatra │→ │ Термо  │
│  Windows, Py      │  │  PDF    │  │ принт. │
└───────────────────┘  └─────────┘  └────────┘
```

## Основные фичи

- Точный фильтр "готов к передаче": `state=KASPI_DELIVERY + status=ACCEPTED_BY_MERCHANT + assembled + kaspiDelivery.courierTransmissionDate=None` — совпадает с числом в Каспи-кабинете
- Сортировка для сборщиков: группа A (1×1 шт), B (1×N шт), C (несколько позиций); внутри — по частоте товара
- Обрезка белых полей на A4-накладных (~2× уменьшение объёма)
- Нормализация всех страниц под 75×120мм (термо-этикетка)
- Разбивка на файлы под рулоны А (полный) и Б (маленький для хвоста)
- **Тестовый режим**: полный PDF собирается, но на печать отправляется только N (default 5)
- Идемпотентность: заказы + статусы в БД, дубликатов между запусками нет

## API

- `GET /` — веб-интерфейс
- `GET /config` — города + дефолты
- `POST /jobs` — создать сборку
- `GET /jobs/{id}` — статус
- `GET /jobs/{id}/tasks` — задачи для агентов
- `GET /jobs/{id}/pdf/{filename}` — скачать PDF
- `GET /agent/next-task?city=X` — агент: следующая задача (auth: Bearer)
- `GET /agent/pdf/{task_id}` — агент: скачать PDF задачи
- `POST /agent/complete/{task_id}` — агент: отчитаться

## Локальный запуск

```bash
python -m venv .venv
.venv/Scripts/activate    # или source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
uvicorn app.main:app --reload --port 8090
# → http://localhost:8090
```
